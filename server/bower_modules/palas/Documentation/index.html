Overview

This template is built with the Bootstrap framework and it uses jQuery, a javascript library to handle some of the actions throughout the template, and in order to be able to easily work with the template, you need to know HTML and CSS, and at least the basics of these two frameworks, but it is not necessary.

However, if you want to make some major changes in the template, some knowledges of the Bootstrap framework (grid system mainly) are required.

Also, we used some third party plugins in this template like isotope, owl-carousel etc (we will enumerate them later in this documentation ) and there may be some situations where you will need or want to make some changes. e.g. If you want some new carousels in your site, you'll need to initialize the owl-carousel.js plugin, and therefore you have to know how to do that. But, don't worry, we will explain what every plugin does and where you could get more informations about them. So, let's get started.

Files Structure

Once downloaded, unzip the compressed folder to see the structure of the template. Plugins folder includes all third party plugins while css, js and php folders includes only files created by us.

We strongly advise you not make your customizations in these files, as you may need to replace these files in the future updates. If you want to add something to your site, or if you want to change something (styles or scripts), please put your code in separated files (e.g. user_style.css).

HTML Structure

This template is based on a responsive layout and uses the boostrap grid system. The general structure is the same throughout the whole template.

When you modify or create a page, make sure you have the required files included (plugin dependencies).

Open some HTML files and take a look at some sections and try to make an idea of how this template is built, analyze the code for a little bit.

The whole content is wrapped in a #wrapper.

For Home pages, the page usually starts with a header, then the slider, then some sections (services, about, team etc) and it ends with the footer.

For blog, shop, portfolio pages the page starts with the header, then an intro section (Title of the page and breadcrumbs), then a whole section page (content) and it ends with the footer.

The css files are loaded in the head tag and the javascript code is loaded at the bottom of the page and we recommend to let them there (performance benefits).

The HTML files are (we say) commented enough to understand the code. As I said at the beginning, it is important to take a look for a few minutes at some HTML files to analyze first how the elements are placed. Analyze the header (navigation menu), the slider and some home page sections and you'll have no problem changing the code.

It is important to know the basics of the Bootstrap Grid System .

CSS Structure

We have a single css file that handles all the content. It seems to be huge, but we created a table of content that helps you to find, faster and easier, every class name you want. Open style.css file and at the beginning you will se the structure of the content, the main colors used in the template, the main fonts and some notes.

To easily find a class name, you could use CTRL+F in any text editor.

Tip: If your text editor suports to "minimize a section", we included some comments (and the code is pushed with a tab) to easily minimize a section, and work with a file faster.

Javascript Structure

The javascript structure is almost the same like the css tructure. We have a single file that handles all the content. Open script.js and at the beginning you will see the structure of the content.

Sections

Header - Navigation Menu

The best way to create a navigation menu is to copy the desired header from a home page and then change the content.

Useful classes:

Add static-menu class to header-wrapper to have a static menu (not fixed) (index_03).

Add menu-light class to menu to have a light menu (white) (index_03).

Add menu-dark-transparent class to menu so that the menu will be a little transparent (index_06).

Logo

Note that we didnt use an img tag for the logo image, but a background-image property because there are moments when you will need a light and a dark logo (light for desktops and dark for mobiles, or light for transparent Header and dark for the sticky menu - when you scroll, the logo changes from light to dark). The easiest way to add a logo, is to name the light version logo-light.png and the dark version logo.png and place them in images folder, replacing the default ones. This way you will not need to change more css properties from style.css.

Helper Classes

For a better and quicker development we created some "helper classes". Throughout the HTML files you will find some classes like mb20 mt0 pb0 color-on-dark. These classes are pretty self-explanatory and I will not explain them here one by one. You will find them in style.css file "Helper Classes" Section.

Mainly, we used classes like mb50 pt0 (margins and paddings) to solve some inconsistencies between sections. For example, if you want two background sections side by side, you will have to add mb0 class (margin-bottom:0px) to the first one and mt0 class (margin-top:0px) to the second one, because background sections (section-bg) have a vertical margin of 40px. This way you could fix some margin or padding issues without the need to create a new class or anything else.

Grid System

In order to easily work with the template you should know the basics of the Bootstrap Grid System. Please take a look at it before you make any changes related to containers, rows, columns etc in the template.

We want to explain some classes we used in the template that are related to a grid system.

row col-p0 - No paddings between columns (useful for columns with a background)

row col-p30 - 30px padding left and right between columns

Similar classes: col-p5, col-p10, col-p20. Add these classes to a row (class) to increase or decrease the padding between columns (15px padding is default)

col-*-* xs-box On mobiles, it will add a bottom padding of 30px (The default grid system doesn't add any margin when a column is pushed below)

col-*-* sm-box On tablets, it will add a bottom padding of 30px

Similar classes: xs-box2, xs-box3, sm-box2, sm-box3 (box2 is 40px, and box3 is 50px) (bottom padding)

Background images

If you want to change the background image of a section you will have to open style.css (or user_style.css) and change the bg01 class (or bg02, bg03, bg14 etc) with the path to your desired image.

bg-img class is required (it sets some background properties)
bg03 class is the path to the image
bg-overlay class is for light images (When you want a dark overlay background). Recomend to include it sometimes to have a dark overlay and change the opacity (op6, op4, op8 etc) so that the content inside to be visible and readable (when the text is white)
stellar class is for parallax backgrounds (data-stellar-background-ratio is required)

Contact Form

This template contains a fullys secure working contact form. All you need to do is to replace some lines of code. So, in order to receive messages you need to edit contact.php file from php folder and type in your own email address and your secret google captcha key (more info below).

Please ensure that PHP mail() function is correctly configured on your server.

Google Captcha

1. Go to https://www.google.com/recaptcha/admin (if you're not logged in, please log with a google account) and click the blue Get reCAPTCHA button (top right).

2. Set a label and the domain you want to use the captcha.

3. Copy the secret key and paste it to contact.php (image above), then copy the site key and paste it to the contact form from page_contact.html (image below).

Subscribe Form

This template includes a fully working MailChimp Subscribe form and there are some steps you need to follow to make it work.

Firstly, if you already know about MailChimp, you have an account and know how to obtain the form action link, all you have to do is to replace it in the subscribe form of the template.

Now, if you don't have an account, go to http://mailchimp.com/, read its features and see if you really need it. If you don't like, or you prefer other Email Marketing company, you have a similar form template (the same style without MailChimp id's and name's) in this documentation - code/simple_subscribe_form.html. Replace the Mailchimp form in the template with this one and integrate you're desired email marketer.

If you like mailchimp, create an account, create a list and so on. Details about how to create an account and make it functional, you'll find on the mailchimp knowledgebase.

Where to get the form action link?

Log in to mailchimp, go to Lists (assuming that you created a list), click on the desired/created list, click on the Sign up Forms button from the menu, select Embedded Forms, and in the Copy/paste onto your site section, copy the action link and paste to you form in the template as we showed in the image above.

Twitter Feed

Although we didn't include a working twiter feed widget in our preview (we didn't want to decrease the performance) we used the tweetie.js plugin so that our customers can have a working twitter feed widget with our template. We initialized the tweetie.js to match our template, but you have to change some php code to make it work. (We'll explain later)

Firstly, instead of the dummy tweeet content from the template, include <div class="twitter-feed"></div>.

Don't forget to insert the plugin at the bottom of the page you want the widget to run ( <script src="plugins/tweetie/tweetie.min.js"></script> )

Now, you need the consumer key and the secret key. Go to https://apps.twitter.com/, log in if you're not, then include the requested details (label for the app, the site you want the widget to work etc). If all is good, click on the Keys and access tokens and from there you should be able to copy the secret keys and insert them in plugins/tweetie/api.config.php (4 variables.)

We tested the plugin with our secret keys and it's working. If you have encounter any troubles, take a look at the tweeti.js github page. It exaplains in more details what you should do.
